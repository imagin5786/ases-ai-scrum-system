ases-hook.py — CARL-Inspired
4 JOBS · PreToolUse
Job 1 — Context Injection
Builds <ases-state> block from context.json (always) + sprint_context.json (active sprint). Resolves relevant_decisions[] IDs to full ADR text. Writes injection to .ases/.current_injection.md.
Trigger: any ases-* command
Job 2 — PO-Only Guard
Blocks Read calls to .ases/decisions.json, global_context.json, docs/prd.md, docs/roadmap.md, docs/hld.md unless the active command is PO-facing. Agents get file-level access control.
Trigger: Read tool on protected files
Job 3 — Commit Guard
Blocks git commit if uat_report.verdict ≠ APPROVED | CONDITIONAL, or if current_phase ≠ SPRINT_SHIP. Enforces the "commit only after UAT" rule at system level. Replaces guard_commit.py.
Trigger: Bash git commit
Job 4 — UI Scaffold Guard
Blocks Write to /ui/ paths not declared in ui_scaffold_manifest integration_points[]. Gemini's scaffold is locked. Replaces guard_ui_scaffold.py. Combined with Jobs 1-3 in single hook.
Trigger: Write to ui/**
Context Bracket Injection
ADAPTIVE VERBOSITY
FRESH >70%
Full injection. Level 3 + Level 2. Sprint goal, task status, open issues. Resolved ADR text from relevant_decisions[].
MODERATE 40–70%
Abbreviated injection. Level 3 + sprint goal + task counts + ADR IDs only (not full text). Saves ~40% of injection tokens.
DEPLETED 15–40%
Minimal injection. Level 3 only. Sprint ID + task complete/total. No ADRs. Skills read what they need explicitly.
CRITICAL <15%
Warning only. Injects single line: ⚠ CONTEXT CRITICAL — run /compact before continuing. No state injection.

Human Approval Gates
6 PER SPRINT
Project Start
After /ases-roadmap
PO approves sprint allocation — governs entire project. /ases-init blocked until approved.
Sprint Design
After /ases-sprint-gate PASS
PO approves design docs. sprint_context.json written. Phase 2 locked until confirmed.
Sprint Execution
Critique ESCALATE
Iteration cap reached or spec ambiguity. PO resolves before dev resumes.
Sprint Ship
/ases-uat — PO reviews system
PO reviews running system against every AC item. Rejected items → surgical re-entry. Blocks /ases-devops.
Sprint Ship
After /ases-final-audit SHIP
PO approves audit report. /ases-release blocked until confirmed. Writes global_context entries.
Any Phase
BLOCK verdict
Any BLOCK from sprint-gate or final-audit routes to PO with exact surgical re-entry path.
